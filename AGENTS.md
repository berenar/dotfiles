# AGENTS.md

This file provides guidance for AI coding agents working in this repository.

---

## Repository Overview

This is a **macOS machine setup repository** containing:

- **Dotfiles** managed with [GNU Stow](https://www.gnu.org/software/stow/)
- **Packages** via Homebrew (Brewfile)
- **System preferences** via shell script

This is a **config-only repository** - there are no build steps, tests, or linting to run.

---

## Quick Reference Commands

### Full bootstrap on a new machine

```bash
bash scripts/bootstrap.sh
```

This installs Homebrew, runs `brew bundle`, stows every package under `dotfiles/`, fixes `~/.ssh/config` perms to `600`, restarts the tmux server, and applies macOS defaults.

### Package Management

```bash
# Install all packages from Brewfile
brew bundle

# Regenerate Brewfile from currently installed packages
brew bundle dump --force --no-restart --no-vscode
```

### Dotfiles Management

Stow packages are located in the `dotfiles/` subdirectory.

```bash
# Apply a dotfile package (creates symlinks to home directory)
stow -d dotfiles -t ~ <package>

# Remove a dotfile package
stow -d dotfiles -t ~ -D <package>

# Preview what stow would do (dry run)
stow -d dotfiles -t ~ -n -v <package>
```

### macOS Configuration

```bash
# Apply macOS system preferences
bash scripts/macos-config.sh
```

## Stow Package Structure

Each stow package is a directory inside `dotfiles/`. The internal structure mirrors
the home directory layout.

### Current Packages

agents, claude, codex, finicky, git, kitty, lazydocker, lazygit, nvim, opencode, ssh, tmux, zshrc

### Adding New Dotfiles

To add a new config file to the repository:

1. Create a directory inside `dotfiles/` matching the application name
2. Inside, recreate the path from home directory
3. Move the config file into this structure
4. Run `stow -d dotfiles -t ~ <package>` to create the symlink

Example: Adding `~/.config/foo/config.yml`

```
dotfiles/
  foo/
    .config/
      foo/
        config.yml
```

### Finicky

After editing `.finicky.js` in this repo, restart Finicky to pick up changes —
the stowed symlink breaks its file watcher:

```bash
pkill -x Finicky && open -a Finicky
```

## File Operations

### Deleting Files

Use `trash` instead of `rm` for safety (allows recovery):

```bash
# Preferred
trash ~/.config/old-app

# Only for dependencies
rm -rf node_modules
```

### Protected Files

Never modify these files directly:

- `.env` files (anywhere in the repo)
- `.zsh_secrets` (contains private tokens/keys)

## Agent-Agnostic Config (`dotfiles/agents/.agents/`)

Everything shared between coding agents lives in the `agents` stow package,
following the `.agents` cross-agent convention (stowed to `~/.agents/`):

```
dotfiles/agents/.agents/
  skills/          # Agent Skills (SKILL.md format) - one dir per skill
  commands/        # Slash commands / prompts (markdown)
  instructions/    # Global system instructions (markdown)
```

How each agent picks these up:

| Agent       | Skills                          | Commands                                  | Instructions                               |
| ----------- | ------------------------------- | ----------------------------------------- | ------------------------------------------ |
| Claude Code | `.claude/skills` symlink        | `.claude/commands` symlink                | `@~/.agents/instructions/*` in `CLAUDE.md` |
| OpenCode    | reads `~/.agents/skills` native | `.config/opencode/command` symlink        | `instructions` array in `opencode.jsonc`   |
| Codex       | reads `~/.agents/skills` native | `.codex/prompts` symlink                  | generated `~/.codex/AGENTS.md` (see below) |

New skills and commands added under `.agents/` are picked up by all agents
automatically (restart the agent — they load at session start). No per-item
symlinks are needed; the `skills`/`commands` directories are symlinked whole.

Codex has no include syntax for `AGENTS.md`, so `dotfiles/codex/.codex/AGENTS.md`
is generated by concatenating the instruction files. After editing anything in
`.agents/instructions/`, regenerate it:

```bash
bash scripts/build-codex-agents-md.sh
```

Command files use frontmatter keys from both agents (`description`, `agent`);
each agent ignores keys it doesn't know. Keep command bodies agent-neutral.

## OpenCode Plugin Development

The `dotfiles/opencode/.config/opencode/` directory contains TypeScript plugins.

### Plugin Structure

```
dotfiles/opencode/.config/opencode/
  plugin/           # TypeScript plugins
  command/          # Symlink to ../../agents/.agents/commands
  opencode.jsonc    # Main configuration (instructions point to ~/.agents/instructions)
```

### Plugin Style (TypeScript)

When modifying plugins:

- Use `async/await` over promise chains
- Use named imports: `import { foo } from "bar"`
- Use `const` over `let`
- No `any` type - use proper types or `unknown`
- No code comments
